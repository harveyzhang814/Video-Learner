Gemini CLI 是为需要处理本地文件和调用多种工具的任务而设计的终端代理工具。本课程将深入讲解 Gemini CLI 的工作原理，并探讨命令行相比 Web 界面所带来的强大优势。希望本课程能启发你对 Gemini CLI 无限可能的想象。

## Gemini CLI 简介 [00:01:24]

### 什么是 Gemini CLI

Gemini CLI 是一个运行在终端中的智能代理工具。它安装简便、极为轻量，用户可以直接通过提示词提问，代理会代替你进行研究和探索。Gemini CLI 能够定位并读取所需的文件，理解内容后给出清晰的回复。

从本质上讲，Gemini CLI 是一个对话式 AI 代理，旨在作为交互式助手使用。用户可以与代理进行多轮对话，共同完成各种任务。

### Gemini CLI 的特点

- **开源性质**：Gemini CLI 是开源项目，由社区驱动功能开发。用户可以检查每一行代码，进行自定义修改，分支代码添加自己的功能，或创建专属版本的 Gemini CLI。

- **Gemini 模型驱动**：顾名思义，Gemini CLI 由 Gemini 模型提供支持和驱动。免费版本提供了相当慷慨的请求配额，可以免费用于日常工作。

## Gemini CLI 工作原理 [00:02:26]

理解 Gemini CLI 的内部工作机制：

### 提示词处理流程

1. **发送提示词**：你发送一个提示词给 Gemini CLI
2. **推理理解**：提示词被发送到大型语言模型 Gemini 模型，进行推理以理解需要哪些工具
3. **工具调用**：如果需要处理信息或向你请求后续信息，代理会按顺序调用工具以生成正确的响应
4. **迭代循环**：这个过程可以多次迭代，代理通过调用不同的工具、获取不同信息并进行推理，直到对响应满意后才返回给你

这种设计非常强大，意味着 Gemini CLI 可以长时间运行，进行推理和循环调用各种工具，帮你构建完整的应用程序或调试复杂的问题，让你专注于自己最擅长的创造工作。

## 终端代理的优势 [00:03:17]

相比 Web 界面，使用终端代理具有以下主要优势：

### 直接访问文件系统 [00:03:21]

终端代理可以直接访问你机器上的所有文件。Gemini CLI 可以读取回答问题所需的文件，这种上下文感知能力使其非常具有对话感，让它像一个任务伙伴一样。

### 工具百宝箱 [00:03:40]

像 Gemini CLI 这样的终端代理就像一把瑞士军刀。你本地计算机上安装的任何工具，Gemini CLI 都可以访问并为你安装。例如，如果你要部署应用程序或查看已部署服务的日志，可以让 Gemini CLI 使用 G-Cloud 来拉取这些日志。这真正帮助你避免了上下文切换，因为 Gemini CLI 可以代表你安装工具并执行操作。

### 自动化与脚本 [00:04:05]

终端代理非常适合自动化和脚本编写任务。不需要去学习如何做某件事，你可以直接让 Gemini CLI 构建一个脚本，然后为你运行它。

### 扩展生态系统 [00:04:13]

Gemini CLI 拥有庞大的扩展生态系统。通过添加扩展、MCP 服务器或自定义命令，可以完全自定义 Gemini CLI 来完成任何任务。

## Gemini CLI 内置工具 [00:04:29]

### 文件系统工具

在基础层面，Gemini CLI 附带了一系列内置工具，包括：列出目录、读取文件、写入文件、搜索和编辑。

### 网页搜索工具

Gemini CLI 另一个很酷的功能是具备网页搜索工具。当 Gemini CLI 需要访问刚发布的内容、新闻或是你希望它研究未被训练过的最新数据时，这个功能非常有用。

### Shell 工具

如前所述，Gemini CLI 还具备 Shell 工具，可以通过 Shell 命令运行你机器上的任何应用程序。例如，如果你有 GitHub CLI，它可以运行 Shell 命令来执行 GitHub CLI 并为你创建 Pull Request。

### 记忆功能

还有其他工具，比如保存记忆的能力。当你经常使用 Gemini CLI 时，它可以记住你自定义工作流程的偏好，或是某些你不想在每个会话中重复的细节。
