# 贡献指南

## 开发环境初始化

克隆仓库后，运行以下命令安装 Git hooks：

```bash
bash hooks/install.sh
```

hooks 存放于 `hooks/` 目录，会被软链接到 `.git/hooks/`。目前包含：

| Hook | 作用 |
|------|------|
| `pre-commit` | 禁止直接在 `master` / `staging` 上提交（只允许合并提交） |

每次新增 hook 后重新运行 `bash hooks/install.sh` 即可。

## 提问与问题描述规范

向 Claude Code 或其他协作者描述问题时，避免意图含糊的描述。以下示例说明如何将模糊问题精炼为可操作的技术 spec。

### 典型错误：意图含糊 / 上下文缺失

只描述"界面某处有问题"，但缺少：
- 页面/组件路径
- 期望值与当前值
- 复现步骤
- 明确的约束类型（视觉感受还是像素限制）

### 示例：UI 宽度被限制

**模糊描述**：
> GUI 的播放界面（右边栏）的宽度有上限，太小了，查看容器和子容器，是什么制约了宽度

**精炼后的描述**（可直接复制使用）：
```text
请帮我定位 Electron GUI 里右侧播放栏（video panel）的宽度为什么有"最大值"限制：
- 入口文件/页面：electron/src/renderer/index.html
- 现象：拖动中间的竖向 resizer 时，右侧栏宽度最多只能到 <xxx> px（或"明显到不了更宽"）
- 期望：最大宽度至少允许到 <yyy> px，或跟随窗口宽度自适应
请指出具体是哪条 CSS 或 JS clamp/计算逻辑在限制（给出代码行号范围），以及如何调整最大值/改为可配置。
```

**好在哪里**：
- **可直接定位到规则来源**：明确入口文件与交互（resizer），能快速找到 `--video-panel-width` 与 `clampVideoPanelWidth()` 之类的限制点。
- **把"问题"转成"可验证的约束"**：用当前值/期望值描述，方便给出精确修改方案（改 max、改成配置、或改成百分比/自适应）。

### 精炼 query 的通用模板

```text
请帮我定位 <组件/功能> 中 <现象>：
- 入口文件/页面：<文件路径>
- 现象：<当前行为，越具体越好>
- 期望：<预期行为>
请指出具体是哪 <行/条/函数> 在导致这个问题（给出代码行号范围），以及如何修改。
```
