# md2subtitle.py 修复设计方案

## 问题分析

`md2subtitle.py` 存在以下问题：

1. **忽略原始时间范围**：使用固定 3 秒作为 duration，应该使用 `original.md` 中的实际 `--> ` 时间范围
2. **VTT 格式错误**：每行生成两条 cue（一条空时间，一条带文本），违反 VTT 规范
3. **未过滤无效条目**：没有跳过"没有字幕的时间段"

## 修正方案

### 1. 修改 `parse_original_md` 函数

- 匹配格式：`[hh:mm:ss] HH:MM:SS.mmm --> HH:MM:SS.mmm 文本`
- 提取并保留 `start_time` 和 `end_time`
- 过滤无效条目：
  - 时间范围无效（start >= end）
  - 文本为空

### 2. 修改 `convert_to_vtt` 函数

- 使用实际的 `start_time` 和 `end_time`
- 每个 entry 只生成一条 cue

### 3. 修改 `convert_to_srt` 函数

同上，保持一致性

## 修正后的输出格式

### VTT
```
WEBVTT

1
00:00:03.000 --> 00:00:04.500
又来搅局了一个赛道

2
00:00:06.166 --> 00:00:07.800
人们想要一个通用Agent
```

### SRT
```
1
00:00:03.000,000 --> 00:00:04.500,000
又来搅局了一个赛道

2
00:00:06.166,000 --> 00:00:07.800,000
人们想要一个通用Agent
```

## 数据结构变更

```python
# 之前
entries = [(seconds, ms, text)]

# 之后
entries = [(start_seconds, end_seconds, start_ms, end_ms, text)]
```

## 测试用例

| 输入 | 期望输出 |
|------|---------|
| `[00:00:00] 00:00:03.000 --> 00:00:04.500 又来搅局了一个赛道` | 生成 cue: 00:00:03.000 --> 00:00:04.500 |
| `[00:00:00] 00:00:03.000 --> 00:00:03.000 无文本` | 跳过（时间无效或空文本） |
