# GUI 人工测试计划（弥补自动化未覆盖流程）

> 与 `2026-03-12-gui-test-automation-design.md` 互补：自动化覆盖 Main/Preload/ServiceClient 与状态逻辑，**本计划仅覆盖需真实 Electron 窗口、视觉与端到端交互**的用例。执行前请先跑通 `npm run test:gui`。

---

## 1. 范围与目标

- **范围**：真实启动 Electron、窗口首屏与可操作性、在 GUI 中的点击/输入/视觉校验、冷启动与退出、端到端「新建任务 → 界面看到步骤与日志」。
- **目标**：在发版或重要合并前，用人工执行一遍关键路径与回归项，确保自动化无法覆盖的「可见行为」正确。
- **不覆盖**：M/P/R 及纯状态逻辑（已自动化）；CLI 模式；未通过 HTTP 暴露的功能（如打开文件夹、删除任务）的完整 E2E。

---

## 2. 环境与前置

- 从仓库根执行：`npm start` 或 `bash start-electron.sh`（确保已 `npm install`）。
- 准备 1 个可访问的 YouTube URL（用于新建任务）。
- 可选：清空或备份 `work/database.sqlite` 以复现「空状态」或「首屏无历史」。

---

## 3. 人工测试用例

### I1 — 冷启动到可操作（P0）


| 步骤  | 操作                                    | 预期                                        |
| --- | ------------------------------------- | ----------------------------------------- |
| 1   | 关闭已打开的 Electron（若有），从终端执行 `npm start` | 无报错退出                                     |
| 2   | 观察窗口出现时间                              | **10s 内**窗口出现，无长时间白屏                      |
| 3   | 观察首屏内容                                | 左侧为**任务列表**（有历史则展示列表项，无则空状态）；无未捕获异常、无整屏报错 |
| 4   | 打开开发者工具（若可）查看 Console                 | 无持续刷红错误；可有 `[agent-http] ready` 等正常日志     |


**通过标准**：窗口及时出现、首屏为列表或空状态、无白屏/崩溃。

---

### I2 — 新建任务端到端（P0）


| 步骤  | 操作                                                     | 预期                                                                  |
| --- | ------------------------------------------------------ | ------------------------------------------------------------------- |
| 1   | 点击「新建」或等价入口                                            | 出现新建任务表单/弹窗                                                         |
| 2   | 输入有效 YouTube URL，填写 FOCUS（可选）、选择 mode（如 transcript），确认 | 提交后列表或详情区出现**新任务**                                                  |
| 3   | 观察该任务的步骤区域（pills 或步骤列表）                                | 初始为 **pending**，随后有步骤变为 **active/running**，最终有 **done** 或 **error** |
| 4   | 打开该任务的日志区域（若有弹窗或面板）                                    | 有**日志行输出**，随 step 运行增加；错误行可区分（如红色或图标）                               |
| 5   | 等待至少一个 step 完成                                         | 对应 pill/步骤状态变为 done 或 error，与日志内容一致                                 |


**通过标准**：新建成功、步骤状态在界面上正确变化、日志有输出且与步骤对应。

---

### I3 — 关闭应用无残留（P0）


| 步骤  | 操作                                         | 预期                              |
| --- | ------------------------------------------ | ------------------------------- |
| 1   | 正常关闭所有窗口或使用「退出」                            | 应用完全退出                          |
| 2   | 在终端执行 `ps aux | grep -E "node.*http-server | electron"`（或` lsof -i :<曾用端口>`） |


**通过标准**：退出后无僵尸 node 进程、端口释放。

---

### U1 — 首屏加载（视觉）（P0）


| 步骤  | 操作              | 预期                          |
| --- | --------------- | --------------------------- |
| 1   | 启动应用（若已有历史任务）   | 左侧列表展示历史任务（taskId/标题/状态等可见） |
| 2   | 若无历史（空 DB 或新环境） | 显示**空状态**提示，且有「新建」等可点击入口    |


**通过标准**：首屏要么是列表要么是空状态，无空白/卡死。

---

### U2 — 任务列表与选中（P1）


| 步骤  | 操作                   | 预期                            |
| --- | -------------------- | ----------------------------- |
| 1   | 存在至少 2 个任务时，点击列表中某一项 | 该项高亮/选中                       |
| 2   | 观察右侧或详情区             | **刷新为当前选中任务**的详情（meta、步骤、日志等） |


**通过标准**：点击列表项后详情区与选中任务一致。

---

### U3 — 新建任务（界面流程）（P0）


| 步骤  | 操作                   | 预期                                |
| --- | -------------------- | --------------------------------- |
| 1   | 点击新建，输入 URL 与可选参数后确认 | 列表或详情中**立即或短时间内**出现新任务            |
| 2   | 观察新任务步骤              | 步骤 pills 为 **pending**，随后随服务端事件更新 |


**通过标准**：与 I2 中「新建」部分一致；新任务在界面可见且步骤初始为 pending。

---

### U4 / U7 — SSE 与步骤 Pills 在界面上的表现（P1）


| 步骤  | 操作                 | 预期                                                                               |
| --- | ------------------ | -------------------------------------------------------------------------------- |
| 1   | 新建任务或选中一个运行中/未完成任务 | 步骤区域展示 fetch / video / audio / subs / vtt2md / md2vtt / article / summary 等 pill |
| 2   | 等待或观察步骤变化          | pill 状态随 step 变化：**pending → active → done / error**，与日志或服务端一致                   |


**通过标准**：步骤 pill 与真实 step 状态一致，无长期卡在某一状态（除非服务端确实未推进）。

---

### U5 — 日志追加与滚动（P1）


| 步骤  | 操作                           | 预期                                  |
| --- | ---------------------------- | ----------------------------------- |
| 1   | 打开某任务的日志面板/弹窗，且该任务有 step 在运行 | 日志区域有**新行追加**                       |
| 2   | 观察滚动与重复                      | 自动**滚动到底部**（或可手动滚到底）；同一 seq 不重复出现多行 |
| 3   | 若有错误输出                       | 错误行有**高亮或图标**区分                     |


**通过标准**：日志实时追加、去重、可读，错误可见。

---

### U6 — Resync 表现（P1，若可观测）


| 步骤  | 操作                   | 预期                                             |
| --- | -------------------- | ---------------------------------------------- |
| 1   | 在长时间断线重连或服务重启后，选中某任务 | 任务/步骤状态能**与服务器一致**（通过 getTask 或快照刷新），不长期显示过期状态 |


**通过标准**：重连或异常后界面状态能恢复一致（若当前实现有 resync 逻辑）。

---

### U8 — 空状态（视觉）（P1）


| 步骤  | 操作                      | 预期                                 |
| --- | ----------------------- | ---------------------------------- |
| 1   | 在无任何任务时打开应用（或清空 DB 后重启） | 显示**空状态**提示（如「暂无任务」），**新建**按钮或入口可用 |


**通过标准**：空列表时提示明确、新建可用。

---

### 附加 — 控制台无 token 泄漏（P0）


| 步骤  | 操作              | 预期                                                        |
| --- | --------------- | --------------------------------------------------------- |
| 1   | 启动应用，查看终端/控制台输出 | 所有输出中**不出现**明文 `?token=` 后的真实 token，应为 `[REDACTED]` 或等价脱敏 |


**通过标准**：与自动化 M4 一致，人工确认控制台脱敏。

---

## 4. 非目标（占位，后续可补充）

- **打开文件夹**、**删除任务**：若功能尚未通过 HTTP 暴露或未在 GUI 实现，仅列占位，待实现后再补人工用例。
- **播放器与字幕切换**：若在本次 GUI 范围内，可单独加一节「播放与字幕」人工用例；否则不写入本计划。

---

## 5. 执行建议

- **发版/合并前**：至少执行 **I1、I2、I3、U1、U3** 及「控制台无 token 泄漏」。
- **完整回归**：执行全部上表用例，并在不同环境（如无历史 / 有历史、网络正常 / 异常）各跑一次 I2。
- **记录**：建议用清单（checkbox）记录通过/失败与备注，便于追溯。
