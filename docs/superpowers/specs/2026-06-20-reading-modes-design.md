## 一、背景与目标

当前任务详情页为固定的 42%/58% 左右分栏（播放器+字幕 | 文章），不感知媒体类型，没有切换机制。用户在不同场景下（看视频讲座、听播客、精读文章、做笔记）的注意力中心截然不同，单一布局无法满足。

**目标**：支持五种阅读模式，根据媒体类型自动推荐默认模式，用户可自由切换，切换不中断播放。

---

## 二、五种阅读模式

### 模式 A — 视频优先（默认：有视频内容）

| 区域 | 内容 | 占比 |
|------|------|------|
| 左列 | 视频播放器（字幕叠层）+ 笔记模块 | 55% |
| 右列 | 总结/文章 tab + 阅读器 | 45% |

- 字幕嵌入视频底部叠层，播放控制栏提供「字幕」开关
- 左列下方笔记模块替代原字幕列表

### 模式 B — 阅读优先（手动切换）

| 区域 | 内容 | 占比 |
|------|------|------|
| 主区 | 阅读器 + 笔记栏 | flex-1 |
| 右侧边栏 | 视频播放器 + 字幕列表（可折叠，默认 320px） | 320px |

- 文章行宽 max 720px，笔记栏 260px，合计 1040px 居中
- 侧边栏有折叠/展开切换按钮

### 模式 C — 音频+文章（默认：仅有音频）

| 区域 | 内容 |
|------|------|
| 顶部条 | 紧凑音频播放器（72px 高，全宽） |
| 左列 | 字幕列表（280px） |
| 中列 | 阅读器（720px max）|
| 右列 | 笔记栏（260px） |

### 模式 E — 沉浸阅读（默认：无媒体）

| 区域 | 内容 |
|------|------|
| 全宽 | 阅读器（680px max）+ 笔记栏（260px），合计 1000px 居中 |

- 无播放器，无字幕，header 保持最简
- 最适合专注笔记和精读

### 模式 F — 剧场模式（手动切换）

| 区域 | 内容 |
|------|------|
| 上部 | 全宽视频（max-height 58vh），字幕嵌入叠层，播放控制栏含「字幕」开关 |
| 下部 | 总结/文章 tab + 阅读器（720px max）+ 笔记栏（260px） |

- 无独立字幕列表，字幕完全内嵌视频
- 文章区默认展开（不需用户点击）

---

## 三、媒体类型 → 默认模式映射

| mediaKind | 默认模式 |
|-----------|---------|
| `video` | A 视频优先 |
| `audio` | C 音频+文章 |
| `null`（无媒体） | E 沉浸阅读 |

模式 B、F 为手动触发的进阶模式，不参与自动映射。切换后仅本次有效，下次打开该任务按媒体类型重新推荐。

## 四、笔记模块

### 适用模式

| 模式 | 笔记位置 |
|------|---------|
| A 视频优先 | 左列下方（替代字幕列表） |
| B 阅读优先 | 文章右侧（260px 竖栏） |
| C 音频+文章 | 文章右侧（260px 竖栏） |
| E 沉浸阅读 | 文章右侧（260px 竖栏） |
| F 剧场模式 | 文章区右侧（260px 竖栏） |

### 数据结构（参考）

```ts
interface Note {
  id: string
  taskId: string
  anchor: string        // 段落锚点文本（截断显示）
  mediaTimestamp?: number  // 视频时间戳（秒），可选
  body: string
  createdAt: number
}
```

### 交互

- 选中文章段落 → 快速添加批注（带段落锚点）
- 有播放器时自动附加当前时间戳
- 笔记列表按时间倒序，点击锚点跳转至对应段落

## 五、字幕处理策略

| 模式 | 字幕形态 |
|------|---------|
| A 视频优先 | 视频底部叠层文本，播放栏「字幕」开关控制 |
| B 阅读优先 | 右侧侧边栏字幕列表（跟踪高亮） |
| C 音频+文章 | 左列独立字幕列表（点击跳转） |
| E 沉浸阅读 | 无字幕 |
| F 剧场模式 | 视频底部叠层文本，播放栏「字幕」开关控制 |

## 六、模式切换 UI

- **Header 图标组**：五个布局图标按钮，当前模式高亮（accent 绿色背景）
- **⌘K 命令面板**：支持搜索"切换到视频优先模式"等命令
- **切换动画**：80ms 淡出 → 布局重排 → 淡入，不重载媒体

---

## 七、实现策略

**方案：CSS Grid 区域 + 按需渲染**

- `<Player>` 始终 mount，CSS 控制位置和尺寸，保证播放不中断
- `<SubtitleList>` 有播放器可见时 mount，模式 E 中卸载
- `<Reader>` 始终 mount
- `<VideoSidebar>`（模式 B 侧边栏）仅模式 B 时 mount
- 笔记面板始终 mount，CSS 控制显隐

**Store 变更（`ui-store.ts`）**

```ts
layoutMode: 'A' | 'B' | 'C' | 'E' | 'F'
setLayoutMode: (mode: LayoutMode) => void
```

任务加载后，`tasks.$id.tsx` 根据 `mediaKind` 自动写入默认值。

## 八、文章区宽度约束

| 模式 | 文章 max-width | 笔记宽 | 容器 max-width |
|------|--------------|--------|--------------|
| A | — | —（左列固定宽） | — |
| B | 720px | 260px | 1040px |
| C | 720px | 260px | 1040px |
| E | 680px | 260px | 1000px |
| F | 720px | 260px | 1040px |

最优阅读行宽约 65–75 字符（520–720px），笔记栏填充宽屏留白而不压缩阅读区。
